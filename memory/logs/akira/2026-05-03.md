### [DEBUG] 00:40 マイク権限ダイアログが出ない（真の根本原因）
*Arousal: 0.85*
新バイナリ起動しても macOS のマイク権限ダイアログが一切出ない。
- 比較で発覚: Dock版 (`node_modules/.../electron/dist/Ember Chat.app`) は entitlements 完全に空 + Hardened Runtime 無効 → 制約なくマイクアクセス可。一方 electron-builder 製は Hardened Runtime + 不完全 entitlements で詰みだった
- electron-builder のデフォルト entitlements には `com.apple.security.cs.allow-jit` 等3つしか入っていない。マイク entitlement が無いため、`Info.plist` に `NSMicrophoneUsageDescription` があっても ダイアログは出ない（silent 拒否）
- **対処**: `packages/ember-chat/build/entitlements.mac.plist` 作成 → `package.json` の `build.mac` に `hardenedRuntime: true` + `entitlements` + `entitlementsInherit` 指定 → マイク・カメラ・ネットワーク・ファイル全権限を含めた

### [INSIGHT] entitlements ベースの権限要求
*Arousal: 0.6 | Domain: macos*
Hardened Runtime 有効アプリは entitlements で明示的に許可されてないリソースにアクセスできない。`Info.plist` の `NS*UsageDescription` だけでは macOS は権限ダイアログを出さず silent に拒否する。Dock版で動いて `/Applications/`版で動かない場合は **`codesign -d --entitlements -` で2つを比較**するのが最短診断。

### [INSIGHT] 08:30 真因はヘルパーアプリのアイコン設定不足
*Arousal: 0.95 | Domain: electron+macos*
Akiraさんが `package.json` に `afterPack: scripts/patch-mac-bundle.js` を追加。`Contents/Frameworks/` 配下のヘルパーアプリ（`Ember Chat Helper.app` 等4つ）にも `icon.icns` + `CFBundleIconFile` + `CFBundleName` + `CFBundleDisplayName` を設定するスクリプト。これでビルド → 差し替え → アイコン正しく表示。

**学び**:

- macOS Privacy Settings UI は **ヘルパーアプリ側の設定** を経由してアイコンを識別している
- 比較ベースの診断（Aqua Voice などのシンプル構造アプリと比べる）が最短ルート。私はそこに辿り着くのに時間を浪費した
