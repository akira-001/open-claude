### [ERROR] kotoba-whisper-v2 / v1 が iPad 遠距離音声で 0 segments を返す（構造的）
*Arousal: 0.9 | Emotion: Frustration→Insight*

切替後、`segments=0 lang=ja prob=1.00 duration=*s` で空文字返却が連発。以下を順次試して全部効かず：

1. `chunk_length=15 condition_on_previous_text=False`（公式モデルカード推奨）
2. `compute_type="int8"`（CPU 標準）
3. `no_speech_threshold=0.95 log_prob_threshold=-2.0 compression_ratio_threshold=3.5` 緩和
4. `vad_filter=False`
5. PCM gain 正規化（peak→0.5）
6. kotoba-v1.0 で再試行

**Web 調査で根本原因判明**：Distil-Whisper（kotoba のベース）は anti-hallucination 訓練で「ノイズには空文字」を学習している（1% noise-only training）。kotoba-v2 訓練データは ReazonSpeech（クリーン TV 音声）+ WER>10 サンプル除外 → iPad 遠距離は完全に OOD。**仕様通りの動作**。

→ 結論：パラメータ調整では直らない。コード診断で 30 分以上溶かしたが、**Web 1 検索 5 分で解決**できた。

### [INSIGHT] GitHub OpenAgents の共有ブラウザは Browserbase + Playwright
*Arousal: 0.5 | Emotion: Insight*

`openagents-org/openagents` のコード解析で判明：
- 共有ブラウザはスクラッチではなく **Browserbase（クラウド SaaS）+ Playwright（ローカル）のハイブリッド**
- Cloud モード：Browserbase の `liveUrl` を iframe で表示、Playwright が CDP 経由で操作、persistent context で認証永続
- Local モード：Playwright Chromium headless、スクショ画像で代替（ライブ感弱）
- メッセージ履歴は 3 層ストレージ（RAM 1000 件 / Hot 7 日 / Archive 180 日 gzip）
- エージェント側は MCP `workspace_get_history` 最大 500 件 + Claude Code `--resume` で session 継続

参考価値：Akiraさんの Mei + Eve + cogmem 構成は**意味記憶の質で OpenAgents を上回る**が、認証必須サイトのライブ操作は Browserbase 相当の仕組みが必要。

### [PATTERN] graceful close は 2 段階（browser.close → pgrep kill）
*Arousal: 0.9 | Emotion: Lesson*

最重要発見。`pkill -f "remote-debugging-port=9222"` で殺すと **Google 認証 Cookie（SID/HSID/__Secure-1PSID 等）がメモリから flush されずに消滅**。

**正しい手順**：
1. Playwright で `browser.close()` → CDP 切断と Cookie flush 発動
2. `pgrep -f ... | while read PID; do kill -TERM $PID; done` で残プロセス kill（既に flush 済みなので安全）

注意：
- `browser.close()` だけでは Chromium プロセスは死なない（CDP 切断のみ）
- `pgrep` 戻り値を変数経由で `kill` に渡すと zsh で複数 PID パースエラーになる（`while read` ループ必須）
- SIGKILL（`kill -9`）は flush 機会ゼロで絶対 NG

SKILL.md にコード例 + ❌ パターン明記済み。
