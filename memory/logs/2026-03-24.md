## セッション概要
SlackBot（Mei/Eve）に3つの新機能を実装: ファイルアップロード、cogmem記憶検索、クロスチャンネル横断履歴。共通機能をSHARED_CAPABILITIESに集約しDRY化。

## 実装内容

### 1. ファイルアップロード機能
- `[UPLOAD:/path/to/file]` マーカーでローカルファイルをSlackにアップロード
- Slack API: `files.getUploadURLExternal` → PUT → `files.completeUploadExternal`
- 重複アップロード防止（Set で追跡）
- Slack App に `files:write` / `files:read` スコープ追加済み

### 2. cogmem 記憶検索統合
- 会話ログを cogmem 形式（markdown）で `data/memory/logs/` に出力
- リアルタイムインデックス更新（`cogmem index --file`）
- 毎日20:00に cron でログ変換 + インデックス更新
- 既存JSONL → cogmem変換スクリプト作成済み

### 3. クロスチャンネル横断履歴
- `data/user-history-{userId}.json` に直近20件の全チャンネル会話を保存
- DM/グループチャットのシステムプロンプトに自動注入（~1,500トークン）
- セッション（chatHistory）のディスク永続化、上限20→50ターンに拡張

### 4. SHARED_CAPABILITIES（DRY化）
- 記憶・ファイル共有・インサイトの共通指示を1箇所に集約
- Mei/Eve/チャンネルスレッド全てで同じ定義を参照
- 新ボット追加時は性格定義 + `SHARED_CAPABILITIES` で完結

### 5. UX改善
- 記憶の優先順位: コンテキスト → cogmem search → 聞き返し
- cogmem search 時は「思い出してみる...」等の自然な表現を使用
- 「検索する」「確認する」等のシステム的表現を禁止
