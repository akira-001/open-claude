### [BUGFIX] 16:15 haru — Ember Chat 音声パイプライン全体がブロックされていたバグを修正
*Arousal: 0.8 | Emotion: Discovery*
Whisper サーバーは稼働中なのにダッシュボードが検出できず、Ember Chat の音声出力が一度も動作していなかった。

**根本原因**: `dashboard/server/api.ts` の Whisper 検出が `execSync('lsof -ti :8767')` に依存。launchd 環境で lsof が失敗 → `whisper.running = false` → フロントが iframe を描画しない → ポーリング/TTS パイプライン全体がブロック。

**修正**: HTTP ヘルスチェック (`checkServerAlive`) を一次判定に変更。lsof は PID 取得のフォールバックのみ。

### [BUGFIX] 16:25 haru — Bot 2重起動によるメッセージ処理エラーの根本対策
*Arousal: 0.7 | Emotion: Resolution*
Mei DM にテストメッセージ送信 → 「うまく処理できなかった」エラー。原因は orphan Bot プロセス（4/4 21:30 起動、PPID=1）と PM2 管理 Bot が同時稼働し、Slack イベントがランダムに振り分けられていた。

**根本原因**: Bot 起動時の排他制御が存在しなかった（Scheduler にはロックがあるが Bot 本体にはない）。

**修正**: `src/index.ts` に PID ベースのインスタンスロック（`data/.bot-instance.lock`）を追加。既存プロセスが生きていれば起動を拒否。SIGINT/SIGTERM/exit でロック自動解放。2重起動テストで拒否を確認済み。
