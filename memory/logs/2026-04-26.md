### [ERROR] EP-013 候補: numpy `(audio * scalar)` の非決定的 in-place 化
*Arousal: 0.95 | Emotion: Surprise / Detective Mode*
**パターン**: CPython 3.14 + NumPy 1.26 + 特定の呼び出し履歴下で、`a1 = audio * 32767` の結果が新規バッファではなく audio 自体に in-place で書き込まれる。

**発見経路**:
- ベンチで two_stage_dualvad の CER が異常に悪い（0.97）
- 単独の小さなテストでは再現せず、investigate_dualvad.py の H1（kotoba 5回連続）→ H3（webrtcvad）の流れで再現
- トレース: `STEP1: id_a1=100592eb0`（audio と同じID）→ `audio.mean=-0.000066 → -2.173577`（int16値に上書き）

**影響経路**: webrtcvad が audio を破壊 → 後段の kotoba が異常スケール値を「音声」と解釈 → 幻聴 → CER 0.97

**修正**:
```python
# Before（バグあり）
pcm = (audio * 32767).clip(-32768, 32767).astype(np.int16).tobytes()
# After（dtype 明示で新規バッファ保証）
scaled = np.multiply(audio, 32767.0, dtype=np.float32)
pcm = np.clip(scaled, -32768, 32767).astype(np.int16).tobytes()
```

**検証スクリプト**: `voice_chat/tests/investigate_dualvad.py`

**対策（一般化）**: numpy で float→int 変換などの dtype変更を伴う scalar 演算では `np.multiply(arr, x, dtype=...)` で新規バッファを明示すべき。`arr * x` は環境依存で in-place 化されるリスクがある。

### [INSIGHT] INS 候補: WebRTC VAD は人声 vs メディア人声を原理的に区別できない
*Arousal: 0.85 | Domain: audio-processing*
voice_ratio 分布（27サンプル実測）:
- 静寂・ノイズ: 0.00-0.04（skip可）
- 近接発話: **0.44-0.65**
- 遠距離メディア（人声）: **0.63-1.00**

→ 遠距離メディアの方が voice_ratio が高い**逆転現象**。連続的に人声が流れるメディア（ナレーション・対談）は voice_ratio が連続的に高くなる。WebRTC VAD は「人声成分の有無」しか見ない設計で、近接 vs 遠距離 / 人声 vs メディア人声を区別できない。**閾値による切り分けは原理的に不可能**。

→ 二重VAD（webrtcvad pre-filter）の「メディア弾き」用途は不採用が正しい判定。再設計するなら音響特徴（near-field判定、SNR、duration）と組み合わせる前提が必要。

### [ERROR] EP 候補: numpy/CPython の挙動以外の罠 — 関数内 `from X import Y` の UnboundLocalError
*Arousal: 0.6*
**パターン**: モジュール上部で `from pathlib import Path` 済みなのに、関数内で再度 `from pathlib import Path` を書くと、Python が「Path はこの関数のローカル変数」と解釈し、その関数内の **後続の Path 参照すべて**が UnboundLocalError になる。条件分岐内に書いても、関数全体がローカルスコープ汚染される。
**具体例**: `skills_update_cmd.py` で if 内に `from pathlib import Path` 追加 → 関数下方の `Path.home()` が「未代入のローカル変数アクセス」エラー。
**対策**: モジュール上部の import を再利用する。条件分岐内で `from X import Y` を書く場合は、その import が関数全体のスコープに影響することを意識する。基本的に関数内 import は遅延ロード等の意図がある場合のみ使い、ない場合は上部 import に統一。

### [INSIGHT] dict.get(k, default) は値が空文字列でdefault効かない
*Arousal: 0.7 | Domain: python-pitfall*
`dict.get(k, default)` は **キーが「存在しない」場合のみ** default を返す。空文字列・None・0 は「存在する」値として返却される。設定 JSON で空文字エントリがある場合、フォールバック完全に無効化。`dict.get(k) or default` パターンが正解。レビュー時 `.get(k, "literal")` は要警戒。

### [INSIGHT] 多層フィルタは順に切り分ける
*Arousal: 0.7 | Domain: debugging*
voice_chat ambient は STT → speaker_id → classify_source → decide_intervention → batch low_value → co_view low_conf_skip → tier1 LLM の **7層**を通る。各層が独立した閾値を持つ。1層直しても下の層で死ぬ多段ハマり。本セッションで1層目から順に7壁突破。`memory/voice_chat_ambient_layers.md` に確認順序保存。

### [PATTERN] 状態タイマーの永久延長ループ
*Arousal: 0.6 | Domain: state-machine*
**症状**: ロック・コンテキスト・cooldown 系の state が「一定時間後に解除」設計なのに、実際は永遠に継続する。
**原因**: タイマー `last_X_at` を毎回更新してる（条件成立時に限らず観測のたび更新）→ TTL 永久延長。
**対策**: タイマー更新は **条件確定時のみ**（`media_likely → True` の時に限り `last_at` 更新等）。一般化: 「state を伸ばす操作」と「state を観測する操作」を明確に分離する。
LLM 推論コンテキスト・キャッシュ無効化・キープアライブ等にも同パターンが潜む可能性。
