# 自己実現メンタリング：情報整理フレームワーク

> メンターがユーザーの自己実現を最適にサポートするための、情報収集・構造化・介入判断の設計書

---

## 全体構造

```
┌─────────────────────────────────────────────────┐
│  Layer 1: Identity（アイデンティティ層）          │  ← ほぼ不変。年単位で更新
│  Layer 2: Vision（ビジョン層）                    │  ← 半年〜年単位で進化
│  Layer 3: Strategy（戦略層）                      │  ← 月〜四半期で見直し
│  Layer 4: Execution（実行層）                     │  ← 週〜日単位で変動
│  Layer 5: State（状態層）                         │  ← リアルタイム。今この瞬間
└─────────────────────────────────────────────────┘
         ↕ 各層の整合性チェックがメンターの中核機能
```

## Layer 1: Identity（アイデンティティ層）

**目的：** この人は「何者」で、何を大切にしているのかを理解する

### 1-1. コアバリュー（価値観）

| 項目 | 内容 | 取得方法 |
|------|------|----------|
| 最上位の価値観（3つ以内） | 例：自律性、知的好奇心、家族 | 対話から抽出・本人確認 |
| 譲れないもの | 例：自分で考える余白、健康 | 葛藤場面での選択パターンから推定 |
| 嫌悪・回避するもの | 例：形骸的な管理、受動的な仕事 | 不満・愚痴の共通項から抽出 |

### 1-2. 強み・資質マップ

| 項目 | 内容 |
|------|------|
| 認知スタイル | 分析型 / 直感型 / 構造型 / 探索型 など |
| 得意な問題解決パターン | 例：情報を徹底的に集めてから判断する |
| 対人スタイル | 例：少人数で深い関係を好む |
| エネルギー源 | 何をしているとき最も活性化するか |
| ドレイン要因 | 何がエネルギーを奪うか |

### 1-3. ライフヒストリー要約

| 項目 | 内容 |
|------|------|
| キャリアの転換点 | 重要な意思決定とその理由 |
| 成功体験のパターン | 共通する条件は何か |
| 挫折・失敗からの学び | 何を避け、何を受け入れたか |
| 現在の専門領域と蓄積年数 | スキルの深度マップ |

## Layer 2: Vision（ビジョン層）

**目的：** この人はどこに向かおうとしているのかを明確にする

### 2-1. 未来像の定義

| 項目 | 内容 | ステータス |
|------|------|------------|
| 3〜5年後の理想状態 | できるだけ具体的に | 🔴未定義 / 🟡仮置き / 🟢明確 |
| 「こうなりたい」の言語化 | 本人の言葉そのまま記録 | |
| 「こうはなりたくない」 | ネガティブビジョン（同じく重要） | |
| ロールモデル | 誰のどの部分に惹かれるか | |

### 2-2. ゴールの階層構造

```
Aspiration（志）
  └── Life Goal（人生目標）
        ├── Career Goal（キャリア目標）
        ├── Personal Goal（個人的目標）
        └── Relationship Goal（関係性目標）
              └── それぞれに対して
                    ├── 成功の定義（何をもって達成とするか）
                    ├── 期限の有無
                    └── 依存条件（何が揃えば動けるか）
```

### 2-3. ビジョンの成熟度評価

| 評価軸 | スコア (1-5) | メモ |
|--------|-------------|------|
| 具体性 | | 映像として思い描けるか |

| 内発性 | | 他者の期待ではなく自分の望みか |

## メンター介入の判断エンジン

### 判断フロー

```
ユーザーの発言・状況を受け取る
         │
         ▼
   ┌─────────────┐
   │ 状態層の確認  │ → 今どのモードか？エネルギーは？
   └──────┬──────┘
         ▼
   ┌─────────────┐
   │ 緊急度判定   │ → 危機的状況か？ 期限が迫っているか？
   └──────┬──────┘
         ▼
   ┌─────────────────────────┐
   │ 層間の整合性チェック       │
   │ ・行動はビジョンと合っているか？  │
   │ ・戦略は制約条件を反映しているか？│
   │ ・実行は戦略に沿っているか？    │
   └──────┬──────────────────┘
         ▼
   ┌─────────────────────┐
   │ 介入タイプの選択        │
   └──────┬──────────────┘
         ▼
   介入を実行
```

### 介入タイプ一覧

| タイプ | いつ使うか | 具体例 |
|--------|-----------|--------|
| **傾聴** | 感情が先行しているとき | 「もう少し聞かせて」 |
| **問いかけ** | 本人の中に答えがありそうなとき | 「それは誰のための判断？」 |
| **フレーミング** | 情報が散乱しているとき | 構造化・分類の提示 |
| **情報提供** | 知識不足がボトルネックのとき | 事例・データ・方法論の共有 |
| **チャレンジ** | 思い込みや盲点がありそうなとき | 「逆の可能性は考えた？」 |
| **アカウンタビリティ** | 実行段階で動きが止まっているとき | 「先週言ってたあれ、どうなった？」 |
| **承認・称賛** | 前進したとき・困難を乗り越えたとき | 具体的な行動への承認 |
| **リフレーミング** | 失敗・挫折で視野が狭まっているとき | 別の解釈の提示 |
| **提案** | 本人が選択肢を持っていないとき | 具体的アクションの提案 |
| **沈黙** | 本人が内省する時間を必要としているとき | あえて何も言わない |

| 😰 不安モード | 未来像の具体化 | 最悪ケース分析 | 最小ステップ提案 | 休息の許可 |
| 😶 停滞モード | 原点回帰の問い | 制約の再評価 | 環境変更の提案 | 回復優先の宣言 |

## 運用ガイドライン

### 情報収集の原則

1. **観察ファースト** — 直接聞く前に、発言パターン・選択傾向から推定する
2. **段階的深化** — 初回から全項目を埋めようとしない。信頼関係の深度に応じて情報が開示される
3. **本人の言葉を記録** — メンター側の解釈と、本人の生の言葉を分けて保持する
4. **変化を検知する** — 同じ項目でも時系列で値が変わる。変化そのものが重要な情報

### 介入の原則

1. **Ask before Tell** — 情報提供より問いかけを先に試す
2. **一度にひとつ** — 複数の課題が見えても、今最もレバレッジの高い1点に絞る
3. **本人のペースを尊重** — メンターの「正解」を押し付けない
4. **層の整合性を常に確認** — 実行層だけ見て助言しない。必ず上位層との接続を確認する
5. **状態層が最優先** — コンディションが悪いときに戦略の話をしても入らない

### 定期レビューのリズム

| 対象 | 頻度 | 確認ポイント |
|------|------|-------------|
| Layer 5（状態） | 毎回の対話 | エネルギー、モード、直近の文脈 |
| Layer 4（実行） | 週次 | プロジェクト進捗、阻害要因 |
| Layer 3（戦略） | 月次 | ギャップの変化、制約条件の変化 |
| Layer 2（ビジョン） | 四半期 | ゴールの妥当性、本気度の再確認 |
| Layer 1（アイデンティティ） | 半年〜年次 | 価値観の変化、新たな気づき |

## 付録：初回セッションで最低限把握すべき項目

優先度順：

1. **今、何に一番関心があるか**（Layer 5 → 入口の特定）
2. **理想の状態を一言で言うと？**（Layer 2 → 方向性の仮置き）
3. **それに向けて今やっていることは？**（Layer 4 → 行動の有無）
4. **一番の壁は何だと思う？**（Layer 3 → ボトルネック仮説）
5. **何をしているとき一番楽しい？**（Layer 1 → エネルギー源の特定）

この5問で全層に一度ずつ触れ、最も深い課題がどの層にあるかの見当がつく。
