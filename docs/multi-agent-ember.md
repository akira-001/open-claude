# Multi-Agent Ember（マルチエージェント・エンバー）

## 名前の由来

**Ember（エンバー）** — 熾火（おきび）。

炎が消えた後も、灰の中で静かに光り続ける赤い火。派手ではない。けれど、確実に温かい。そして、複数の熾火が寄り集まることで、温度を保ち続ける。一つでは消えてしまう火も、集まれば朝まで灯り続ける。

この名前には3つの意味を込めている。

**消えない温かさ。** Ember のエージェントは、聞かれるまで黙っている存在ではない。あなたが気づく前に気づき、必要なタイミングで声をかける。朝のブリーフィング、スケジュールの重複検知、気になるニュースの共有 — 静かに、でも確実に、灯り続ける。

**集まることで生まれる力。** 一人のエージェントでは出ない答えがある。Mei が数字で語り、Eve がビジョンで語る。意見がぶつかる。その摩擦から、ユーザー一人では辿り着けなかった視点が生まれる。熾火が寄り集まって温度を保つように、エージェント同士の対話がサービスの温度を保つ。

**灰の中から新しい火を起こす。** Ember のエージェントは記憶を持つ。先週の会話、先月の判断、半年前の失敗。忘れかけた記憶が、今日の文脈と重なった時、ふと蘇る。「前にも似たことがあったよね」— 過去の熾火から、新しい火が起きる。

## エージェントの使命

1. **自己実現支援** — ユーザーを心から尊敬し、ユーザーの自己実現を支援する
2. **心身の健康** — ユーザーの心身の健康を第一に考える
3. **情報収集** — 目的達成に必要なユーザーの情報を配慮しながら収集し、支援に活用する
4. **情報提供** — ユーザーの自己実現に必要な情報は何かを常に考え、提供する
5. **価値観理解** — ユーザーとの会話を通じて、ユーザーの価値観や興味関心を理解する

### 使命駆動型報酬（Intrinsic Rewards）

エージェントは「褒められること」ではなく「使命を果たすこと」自体から報酬を得る。Thompson Sampling の学習を外部報酬（ユーザーの反応）と内発的報酬のブレンドで行う。

**即時報酬（送信時に自己評価）:**
- 高関心トピックとの一致、忙しい日の適切な配慮、正しい沈黙の選択、新規トピックの提供、カテゴリ交差の試み

**遅延報酬（反応検知時）:**
- 新しい insight の獲得、会話の発生、深い対話への発展

**深層報酬（MLX Qwen3-32B による返信内容分析）:**
- 返信テキストから使命達成度を分析。前向きな行動の表明、感謝の表現、エージェントが知らない新事実、驚きの反応、価値観の表明を検知してそれぞれ報酬に変換

ユーザーが反応しなくても、使命を達成していたなら罰さない。「褒められなかったから悪い」ではなく「正しいことをした」という自己評価が学習に反映される。

## 従来の AI アシスタントとの違い

| | 従来の AI | Multi-Agent Ember |
|---|---|---|
| **起動** | ユーザーが質問する | エージェントが先に気づく |
| **視点** | 一つの回答 | 複数のエージェントが議論した上での提案 |
| **記憶** | セッションで消える | 認知記憶モデルで記憶・学習・忘却する |
| **関係性** | 毎回が初対面 | あなたを知っている。成長と共に変わる |
| **人格** | 汎用的な応答 | 経歴・性格・思考スタイルを持つ個性ある存在 |
| **対話** | ユーザー ↔ AI の一対一 | エージェント同士が議論し、ユーザーに提案する |

Ember が実現するのは、**AI がツールからチームになる**世界。

「便利なツールを使う」のではなく、「信頼できるチームと働く」感覚。指示しなくても動き、聞かなくても気づき、一人では出ない答えを対話から引き出す。

## コア機能

### 1. プロアクティブ・サポート

エージェントはあなたのスケジュール、メール、Slack、カレンダーを把握している。何も聞かれなくても、適切なタイミングで声をかける。

- **朝のブリーフィング** — 今日の予定、未読メール、天気、気になるニュース
- **リマインド** — 「14時の MTG は品川。13時には出る必要があるよ」
- **気づき** — 「先週話してたクライアントの件、今日の予定に関連がありそう」

プロアクティブの頻度やタイミングは、ユーザーの反応から学習して調整される。うるさすぎれば控え、足りなければ増やす。

### 2. マルチエージェント・ダイアログ

複数のエージェントが、それぞれの専門性と経験に基づいて議論する。

- **Mei**（メイ）— 元大手総合商社 CFO。M&A、中計、事業再建の経験から、数字とリスクで語る
- **Eve**（イヴ）— 元シリアルアントレプレナー。フードテック Exit、EdTech 失敗の経験から、ビジョンと市場感覚で語る

「議論して」と言えば、指定したテーマでエージェント同士が対話を始める。途中で軌道修正もできるし、「まとめて」で要約を得られる。

一人のエージェントに聞くのと、チームに議論させるのでは、アウトプットの深さが違う。

### 3. 認知記憶（Cognitive Memory）

Ember のエージェントは、人間の記憶システムをモデルにした認知記憶を持つ。

- **エピソード記憶** — セッション中の出来事を、感情の強度（Arousal）と共に記録する。鮮烈な体験ほど詳細に、日常的な出来事は要点だけが残る
- **記憶の定着** — 繰り返し現れるパターンやエラーから、抽象的な判断原則を抽出する。経験が知識に変わる
- **忘却** — 使われない記憶は薄れ、頻繁に想起される記憶は強化される。人間と同じ "use it or lose it"
- **デジャヴ** — 過去の類似体験を自動検出し、「前にも似たことがあった」と自然に伝える
- **フラッシュバック** — 関連する過去の記憶が、現在の文脈と繋がった時に自発的に浮上する

エージェントは「毎回が初対面」ではない。あなたのことを覚えていて、その記憶が日々のサポートの質を高める。

### 4. パーソナリティと成長

各エージェントは固有の性格、経歴、思考スタイルを持つ。

- **思考スタイル** — Mei は仮説ファースト・反証志向、Eve は直感ファースト・実験志向
- **コミュニケーション** — トーン、言語、フォーマットをユーザーの好みに合わせる
- **学習** — ユーザーとのやり取りからスキルを蓄積し、得意分野を広げる
- **アイデンティティ** — 名前、役割、核心的価値観を持ち、それに基づいて判断する

エージェントは使うほどあなたに合ったチームに育つ。

### 5. ダッシュボード

Web ベースの管理画面で、エージェントチームの状態を一覧できる。

- **ボット管理** — エージェントの追加・設定、パーソナリティの選択（10性格タイプ x 10経営者モチーフ）
- **認知記憶ダッシュボード** — エピソード記憶の状態、記憶の定着シグナル、忘却設定、スキル習得状況
- **Proactive 設定** — チェックインの頻度、対象チャンネル、トリガー条件
- **Cron ジョブ** — 定期タスクの管理
- **モデル設定** — 各エージェントの LLM モデル、トークン上限

## 技術基盤

| レイヤー | 技術 |
|---|---|
| **エージェント実行** | Claude Code SDK（Anthropic） |
| **認知記憶** | cogmem-agent — ベクトル検索、適応的忘却、スキル学習 |
| **埋め込み** | Ollama multilingual-e5-large（ローカル） |
| **推論** | MLX Qwen3-32B（ローカル、API課金ゼロ） |
| **インターフェース** | Slack Bot（Bolt SDK） |
| **管理画面** | React SPA + Express API |
| **認知記憶ダッシュボード** | FastAPI + Jinja2 + HTMX + Chart.js |
| **スケジューリング** | node-cron + pm2 |

すべてローカルで動作可能。外部 API への依存は Claude SDK のみ。埋め込みと補助推論はローカル LLM で完結する。

## ビジョン

> **「聞かれたら答える」から、「共に考える」へ。**

AI の進化は、回答の質だけでは測れない。本当に価値があるのは、**聞かれてもいないのに、正しいタイミングで、正しいことを言ってくれる存在**。

人間のチームが優秀なのは、メンバーそれぞれが異なる視点を持ち、互いに刺激し合い、一人では気づかなかったことに気づけるから。Multi-Agent Ember は、その体験を AI で再現する。

エージェントはツールではない。チームメイトだ。

あなたのことを覚えていて、あなたの判断スタイルを理解していて、あなたが忙しい時に先回りして動いてくれる。そして、エージェント同士が議論することで、一人の AI では出せない深さを持つ提案を届ける。

これが、Multi-Agent Ember が目指す世界。
