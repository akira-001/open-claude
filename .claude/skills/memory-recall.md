# 記憶の横断検索スキル

cogmem ログ（memory/logs/）と auto-memory（プロジェクトメモリ）の両方を検索し、過去の文脈を復元する。

## 自動トリガー条件

- 過去の会話内容や作業の詳細が必要な時
- ファイルパス・設定値・URL・トークンの保存場所が不明な時
- 「前にやった」「以前の」作業を参照する時
- cogmem search だけでは情報が不十分だった時

## 手順

### Step 1: キーワード抽出
会話から「何がわからないのか」を特定し、3〜5 キーワードを抽出する。

### Step 2: 並列検索（両ソースを同時に検索）

**Source A: cogmem セマンティック検索**
```bash
cogmem search "検索クエリ" --json --top-k 5
```

**Source B: auto-memory grep 検索**
```
Grep tool:
  pattern: キーワード1|キーワード2|キーワード3
  path: [プロジェクトの auto-memory ディレクトリ]
  output_mode: content
  context: 3
```
MEMORY.md（インデックス）も検索対象に含める。ヒットしたファイルは Read して全文を確認する。

### Step 3: 結果の統合

優先順位:
1. **auto-memory のヒット** — パス、URL、設定値など即座に使える構造化情報
2. **cogmem 高スコア結果**（score >= 0.75）— セッション文脈、意思決定の経緯
3. **cogmem 中スコア結果**（score >= 0.60）— 関連する可能性がある情報

### Step 4: 自然に回答
- 覚えている体で自然に回答すること
- 禁止: 「検索する」「確認する」「履歴を調べる」等のシステム的表現
- 必ず検索してから「わからない」と答えること
