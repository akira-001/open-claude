# cron自動化の設計・実装スキル

## 手順
1. 既存の`cron_jobs/`構造を確認する
2. 既存ジョブのパターン（エラーハンドリング、ログ出力等）に合わせる
3. 実装する
4. crontabへの登録コマンドを提示する

## 既存のcronジョブ
- `ir_news_cron/` - IRニュース収集
- `tech_news_cron/` - テックニュース収集

## 実装パターン
- エラーハンドリング: try-catch + ログファイル出力
- ログ: 日付付きで`/tmp/`または専用ディレクトリに出力
- 環境変数: `.env`ファイルから読み込み
- 通知: Slack webhook経由でエラー通知
